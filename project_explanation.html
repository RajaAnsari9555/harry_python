🔐 Backend Python Auth — Project Explanation

Hinglish mein complete project documentation — PDF ke liye ready

1. Project Kya Hai? (Overview)

Yeh ek FastAPI based Authentication Backend hai jo Python mein likha gaya hai. Iska kaam hai users ko register karna, OTP se verify karna, login karna, aur password reset karna — sab kuch email OTP ke through.

Database ke liye MongoDB Atlas (cloud database) use kiya gaya hai aur emails bhejne ke liye Gmail SMTP use hota hai. Login ke baad user ko ek JWT Token milta hai jo uski identity prove karta hai.

💡 Simple shabdon mein: Jaise kisi app mein aap email se sign up karte ho, OTP aata hai, verify karte ho, phir login karte ho — exactly wahi system yahan bana hai.

2. Technologies Used (Kya Kya Use Kiya)

  • FastAPI — Python ka fast web framework, API banane ke liye
  • MongoDB Atlas — Cloud database, data store karne ke liye
  • Motor — MongoDB ka async driver Python ke liye
  • Pydantic — Request/Response data validate karne ke liye
  • Passlib + bcrypt — Password ko securely hash karne ke liye
  • python-jose — JWT token banane aur decode karne ke liye
  • fastapi-mail — Email bhejne ke liye
  • python-dotenv — .env file se secrets load karne ke liye
  • Uvicorn — FastAPI server run karne ke liye

5. Database Model — User Document

MongoDB mein har user ek document ke roop mein store hota hai. Yeh document user_model.py mein define kiya gaya hai.

Field Type Description
name String User ka naam
email String User ka email address (unique identifier)
hashed_password String Password ka bcrypt hash — plain password kabhi store nahi hota
is_verified Boolean Account verify hua ya nahi (default: False)
otp String / None Current active OTP (use hone ke baad None ho jata hai)
otp_expires_at DateTime / None OTP kab expire hoga (10 minutes baad)
created_at DateTime Account kab bana — automatically set hota hai

6. Schemas — Request Validation

user_schema.py mein Pydantic models hain jo define karte hain ki API ko request mein kya data chahiye. Agar galat data aaya toh FastAPI automatically error return karta hai.

Schema Class Fields Kahan Use Hota Hai
RegisterRequest name, email, password New user register karne ke liye
VerifyOTPRequest email, otp Registration ke baad OTP verify karne ke liye
LoginRequest email Login OTP request karne ke liye (sirf email chahiye)
LoginVerifyRequest email, otp Login OTP verify karke JWT token lene ke liye
ForgotPasswordRequest email Password reset OTP mangwane ke liye

7. API Endpoints (Routes)

Saare routes /auth prefix ke saath hain. Yeh sab POST requests hain.

Method Endpoint Kaam
POST /auth/register Naya user register karo — OTP email pe aayega
POST /auth/register/verify-otp Registration OTP verify karo — account activate hoga
POST /auth/login Login ke liye OTP request karo (sirf email bhejo)
POST /auth/login/verify-otp Login OTP verify karo — JWT token milega
POST /auth/forgot-password Password reset ke liye OTP mangwao
POST /auth/forgot-password/reset OTP verify karke naya password set karo
GET / Health check — API chal rahi hai ya nahi check karo

8. Complete Flow — Step by Step

🟢 Registration Flow

1
User POST /auth/register pe name, email, password bhejta hai.
2
Controller check karta hai — kya yeh email pehle se registered hai? Agar haan toh error return karta hai.
3
Password ko bcrypt se hash kiya jata hai (plain password kabhi save nahi hota).
4
6-digit OTP generate hota hai, 10 minute ki expiry ke saath MongoDB mein save hota hai.
5
OTP user ke email pe Gmail SMTP ke through bheja jata hai.
6
User POST /auth/register/verify-otp pe email + OTP bhejta hai.
7
OTP match kiya jata hai aur expiry check hoti hai. Sahi hone pe is_verified = True ho jata hai. Account activate!

🔵 Login Flow (OTP-based)

1
User POST /auth/login pe sirf email bhejta hai.
2
System check karta hai — user exist karta hai? Account verified hai? Nahi toh error.
3
Naya OTP generate hokar email pe bheja jata hai.
4
User POST /auth/login/verify-otp pe email + OTP bhejta hai.
5
OTP valid hone pe JWT Access Token generate hokar return kiya jata hai. Ab user logged in hai!

🔴 Forgot Password Flow

1
User POST /auth/forgot-password pe email bhejta hai.
2
System user dhundta hai, OTP generate karke email pe bhejta hai.
3
User POST /auth/forgot-password/reset pe email + OTP + new_password bhejta hai.
4
OTP verify hone ke baad naya password hash hokar MongoDB mein save ho jata hai. Password reset complete!

9. Utility Files — Kya Kya Karte Hain

🔑 hash_password.py

bcrypt algorithm use karke password ko hash karta hai. hash_password() plain password leke hashed string return karta hai. verify_password() plain aur hashed password compare karta hai. Yeh isliye zaroori hai kyunki database mein kabhi plain password nahi rakhte.

🔢 otp.py

generate_otp() — 100000 se 999999 ke beech random 6-digit number generate karta hai.
otp_expiry() — current time + 10 minutes ka datetime return karta hai.
is_otp_valid() — stored OTP aur user ka OTP match karta hai, aur expiry bhi check karta hai.

📧 email_service.py

fastapi-mail library use karke Gmail SMTP se email bhejta hai. send_otp_email() function email address, OTP aur subject leke email bhej deta hai. Email body mein OTP aur "valid for 10 minutes" message hota hai.

🎫 jwt_handler.py

create_access_token() — user ka data (email, name) leke JWT token banata hai. Token mein expiry time bhi embed hoti hai (30 minutes).
decode_token() — JWT token ko decode karke original data nikalta hai. Agar token invalid ya expired ho toh empty dict return karta hai.

🗄️ connection.py

Motor (async MongoDB driver) use karke MongoDB Atlas se connection establish karta hai. DATABASE_URL .env file se load hoti hai. users_collection variable globally available hai — saare controllers isko import karke use karte hain.

10. Security Features

  • Password Hashing: bcrypt se hash hota hai — brute force attacks se protection
  • OTP Expiry: OTP sirf 10 minutes valid hota hai — replay attacks se protection
  • OTP Cleanup: Use hone ke baad OTP database se hata diya jata hai
  • JWT Token: Stateless authentication — server pe session store nahi hota
  • Email Verification: Bina OTP verify kiye login nahi ho sakta
  • Environment Variables: Secrets .env mein hain, code mein hardcoded nahi

11. Requirements (Dependencies)

fastapi → Web framework uvicorn → ASGI server (FastAPI run karne ke liye) motor → Async MongoDB driver pymongo → MongoDB base library python-dotenv → .env file load karne ke liye passlib[bcrypt] → Password hashing bcrypt==4.0.1 → bcrypt ka specific version python-jose[cryptography] → JWT token handling fastapi-mail → Email bhejne ke liye pydantic[email] → Email validation support
